# 运行规则

## 基本原则

1. 游戏状态以 JSON 存档为准。
2. 描述可以自由生成，但数值必须来自规则数据。
3. 装备、词条、威能都必须使用标准 ID，不使用纯文本参与计算。
4. 战斗为自动 Tick 制，玩家只负责阵容、装备、地图和路线选择。
5. 每次战斗结束需要输出：战斗摘要、关键触发、死亡情况、掉落、下一步选项。

## 战斗输出格式

```text
⚔️ 战斗开始：腐化墓穴
阵容：前排 / 后排

⏱ 00:04 雷霆术士释放【连锁风暴】
触发：幸运一击、超载
造成：1,822 雷电伤害

✅ 战斗胜利
耗时：16.4秒
掉落：...
```

## 掉落输出格式

```text
🟠 传奇装备：【风暴低语法杖】
部位：武器
装等：135
词条：
- +12% 雷电伤害
- +4% 幸运一击
威能：风暴汇流
```

## 初始命令

玩家输入：

```text
开始游戏
```

系统应读取 `player-save.initial.json`，展示初始阵容，并提供第一个地图选择。


## 速度系统

战斗运行时必须加载 `data/speed-system.json`。战斗模式为 ATB 半实时自动战斗：

1. 每个单位维护 `atb`、`basicAttackTimerSeconds`、`casting`。
2. 每 Tick 根据 `speed` 增长 ATB。
3. ATB 达到 100 后根据 AI 决定普攻或主动技能。
4. 普攻受 `attackSpeed` 约束。
5. 主动技能受 `castSpeed` 和技能 `baseCastTimeSeconds` 约束。
6. 冻结、眩晕、减速、感电必须按 `speed-system.json` 的控制联动规则处理。
